# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in SoWork News, please **DO NOT** create a public GitHub issue. Instead, please report it responsibly by emailing:

**security@sowork-news.com**

Please include the following information in your report:

1. **Description** of the vulnerability
2. **Steps to reproduce** the issue
3. **Potential impact** of the vulnerability
4. **Suggested fix** (if you have one)

We appreciate your responsible disclosure and will acknowledge receipt of your report within 48 hours.

## Security Measures

Our project takes security seriously:

- ✅ All dependencies are regularly updated
- ✅ Code is reviewed before merging
- ✅ We follow OWASP security guidelines
- ✅ Sensitive data is never committed to the repository
- ✅ Environment variables are properly managed

## Supported Versions

Security updates will be provided for:

- **Current version** (latest release): Full support
- **Previous version**: Security patches for critical issues
- **Older versions**: No support

## Security Best Practices for Users

If you're using SoWork News:

1. **Keep dependencies updated** — Run `pnpm update` regularly
2. **Use environment variables** — Never commit secrets to your repository
3. **Review third-party integrations** — Only use trusted AI API keys
4. **Monitor logs** — Check server logs for suspicious activity
5. **Update regularly** — Always use the latest version
